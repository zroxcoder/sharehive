Data & Privacy

Your data is stored locally in your browser using IndexedDB.

IndexedDB is a browser-based database system that stores data directly on your device. This means:

  • ‚úÖ Your data never leaves your computer/device
  • ‚úÖ No servers storing your information
  • ‚úÖ Complete privacy and control
  • ‚úÖ Works without internet after initial load
  • ‚ö†Ô∏è Data is specific to the browser and device you're using
‚ÑπÔ∏è Technical Details

IndexedDB is a standard web technology supported by all modern browsers. It can store text, numbers, files, and structured data efficiently.

Yes! Your data is completely private and secure.

Since ShareHive stores everything locally in your browser:

  • ‚úÖ No data sent to any servers
  • ‚úÖ No analytics or tracking
  • ‚úÖ No third-party access
  • ‚úÖ No data collection of any kind
  • ‚úÖ You have full control
üí° Privacy Best Practice

For maximum security, regularly export your data and encrypt the backup files if they contain sensitive information.

Backing up your data is simple and highly recommended!

Steps to backup:

  1. Go to "‚öôÔ∏è Settings" in the sidebar
  2. Find the "Export Data" or "Backup" section
  3. Click "Export" or "Download Backup"
  4. Save the JSON file to a safe location
  5. Store backups in multiple places (cloud, external drive, email)
‚ö†Ô∏è Critical: Backup Regularly!

Browser data can be lost if you clear cache, uninstall your browser, or switch devices. Create backups weekly or after important updates!

Restoring your data is straightforward:

  1. Go to "‚öôÔ∏è Settings"
  2. Look for "Import Data" or "Restore" option
  3. Click "Import" or "Choose File"
  4. Select your backup JSON file
  5. Confirm the import
  6. Refresh the page to see your restored data
‚ö†Ô∏è Warning

Importing data may overwrite existing data. If you have current data you want to keep, export it first before importing!

Clearing browser data will delete all your ShareHive data!

If you clear your browser's cache, cookies, or site data, IndexedDB will also be cleared, resulting in permanent data loss.

‚ö†Ô∏è Important

This data cannot be recovered unless you have a backup file. Always export your data before clearing browser storage!

Situations that cause data loss:

  • Manually clearing browser cache/cookies/site data
  • Using browser "Reset" or "Clean" features
  • Uninstalling and reinstalling the browser
  • Using incognito/private browsing mode
  • Browser storage quota exceeded

Currently, automatic sync is not available, but you can sync manually.

Manual sync process:

  1. On Device A: Export your data
  2. Transfer the backup file to Device B (email, cloud, USB)
  3. On Device B: Import the backup file
  4. Repeat whenever you need to sync
‚ÑπÔ∏è Future Feature

Cloud synchronization is being considered for future updates, which would allow automatic sync across all your devices!

Storage limits depend on your browser, but typically:

  • Most browsers allow 50-100MB for IndexedDB
  • Some browsers allow up to 1-2GB with user permission
  • Chrome and Firefox tend to offer more storage
  • Mobile browsers usually have smaller limits
‚ö†Ô∏è Storage Tips

Avoid uploading very large files. Use ShareHive for text data and small-to-medium files. Store large files externally and keep links in ShareHive.

Troubleshooting

Common causes of data loss:

  • Browser cache/data was cleared
  • Using incognito/private mode
  • Browser was uninstalled
  • Switched to a different browser
  • Storage quota exceeded
‚ö†Ô∏è Data Recovery

Unfortunately, if IndexedDB data is deleted, it cannot be recovered unless you have a backup file. This is why regular exports are crucial!

Prevention:

  • Export data weekly as backup
  • Store backups in multiple locations
  • Avoid using incognito mode
  • Don't clear site data

Try these solutions:

  1. Delete old, unused data
  2. Remove large uploaded files
  3. Clear browser cache (not site data!)
  4. Close unnecessary browser tabs
  5. Update to latest browser version
  6. Try a different browser
  7. Restart your browser
‚ö†Ô∏è Performance Tip

Large files and thousands of items can slow down ShareHive. Archive old data regularly and avoid storing very large files.

Troubleshooting steps:

  1. Refresh the page (F5 or Ctrl+R)
  2. Clear browser cache
  3. Check if JavaScript is enabled
  4. Try a different browser
  5. Disable browser extensions temporarily
  6. Check browser console for errors (F12)
